- Updated sitemap.md to mark repository create/import as inspected.

Key findings:
- The `/new` form is a compact Primer-style form with owner dropdown, name input, suggestion button, optional description, visibility dropdown, template dropdown, README toggle, .gitignore selector dialog, license dropdown, and green Create repository button.
- Repository names preview a normalized slug when spaces are typed, but the UI still shows an inline warning about allowed characters; the Rust API must perform final validation.
- Visibility is a dropdown menu with Public and Private descriptions, not radio cards in the current GitHub UI.
- `.gitignore` selection is a searchable popup/listbox with many language/framework templates.
- Import creates a new repository, optionally uses source credentials to clone a private Git remote, redirects to an import status page, and emails completion. It imports source code and commit history only; it does not import issues, pull requests, Git LFS objects, or private-network repositories.

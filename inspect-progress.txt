Key findings:
- GitHub docs/API are broad; the first build must prioritize repository hosting, auth/session verification, dashboard empty state, repository creation, REST API foundations, then issues/PRs/actions/packages/search/orgs.
- Target GitHub auth includes email/password, Google, Apple, passkeys, captcha, email verification, and password reset.
- opengithub stack constraint is Better Auth with Google OAuth only. Password, Apple, passkey, captcha, and GitHub OAuth should not be built unless ralph-config.json changes.
- SDK is disabled in ralph-config.json; developer experience should start with REST docs, personal access tokens, Git remote instructions, and webhooks rather than a packaged SDK.
